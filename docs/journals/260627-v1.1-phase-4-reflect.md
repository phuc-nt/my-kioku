# v1.1 Phase 4 — Reflect (living-loop detectors)

**Date:** 2026-06-27 · **Commit:** 5575234 · **Branch:** feat/v1.1-emotional-relations

## What shipped

Three deterministic, read-only reflect detectors over relations/tags — the living-loop signals for the cron agent. This closes the v1.1 loop: agent writes relations → reflect surfaces gaps → agent backfills.

- `findMissingRelations` — strong-mood entries (intensity ≥4 or ≤2) with NO relation row.
- `buildRelationSummary` — top joy/trigger targets per entity (range-scoped).
- `findUnconvertedTags` — tags not matching any entity name (folded) → agent converts tag → link/relation.
- Wired into `suggested_actions` + `--md` (## Emotional relations + ## Tags to convert).

## Key decisions / learnings

- **Field-name discipline verified clean** — reflect.ts returns an inline object; render-markdown.ts has its own structural `ReflectLike`. A field-name drift (`relation_summary` vs `relationSummary`) would silently break `--md` with NO type error. Review confirmed exact match. (The risk the reviewer flagged as highest-value didn't materialize.)
- **M2 (fixed)**: `relation_summary` originally did `GROUP BY r.target` — but targets are stored verbatim, so `[[Chạy bộ]]` and `[[chạy bộ]]` counted as two patterns, undercounting the signal the summary exists to provide. Fixed: fold+merge variants in TS (cheap; `fold` already imported), verified one count=2.
- **M1 (fixed)**: `LIMIT ${TOP_N}` interpolated a constant (safe today) but set a bad precedent in a parameterized-SQL file → bound it.
- **L3 (fixed)**: `missing_emotional_relation` was uncapped while tags capped at 30 — asymmetric at import scale. Capped at 50 (most-recent first).
- **By-design scoping** documented: summary is range-scoped (snapshot); missing-relations + unconverted-tags are all-time (backfill/migration debt isn't period-bound).
- All boundaries verified correct by review: intensity inclusive at 4/2, null-intensity excluded, tag-only entries flagged (tags ≠ relations), any rel_type counts as "has relation".
