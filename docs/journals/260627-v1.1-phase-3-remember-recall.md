# v1.1 Phase 3 — Remember & Recall (relation round-trip)

**Date:** 2026-06-27 · **Commit:** b965d55 · **Branch:** feat/v1.1-emotional-relations

## What shipped

The user-facing layer: relations/tags round-trip through `remember --stdin`; `recall --relation` filter + relation-weighted ranking.

- `remember`: leading `joy::`/`tags::` lines in stdin → indexed relations/tags; relation targets auto-stubbed; result surfaces relations/tags.
- `recall --relation <type>`: hard filter; relation targets get `RELATION_BONUS = 0.5` (> `ENTITY_BONUS = 0.3`); hydrate always includes `relations:{}`/`tags:[]` (stable schema).
- Extracted `hydrate.ts` + `relation-expansion.ts` to keep `recall.ts` under 200 LOC.

## Key decisions / learnings

- **Code review caught a Critical parse-drift (C1)** — `remember` re-parsed `## time\n<text>` (no mood line) to extract relations, but `appendEntry` writes `## time\nmood::\n<text>` to disk. With `--mood` + a leading blank line in stdin, the two parses **diverged**: remember reported `joy::` as a relation, but the on-disk mood line pushed the blank *after* it → field zone ended → index filed `joy::` as **body**. Result: remember lied to the agent, `recall --relation` missed the entry, relation misfiled as a plain link. Reviewer repro'd it end-to-end.
  - **Two-level fix**: (1) `appendEntry` now returns the parsed entry (the indexer's view) — remember never re-parses raw text, so it can't drift. (2) `appendEntry` strips leading blank lines so a blank can't split `mood::` from relation fields. Verified: remember + index now agree.
- **M1 (DRY)**: deleted `matchedEntityNames` (re-implemented entity folding already in `expandByEntity`) — reuse `expansion.entities`, removing a second full-table scan per recall + divergence risk.
- **H1 (documented)**: RELATION_BONUS is additive on FTS, so a relation hit (0.5) outranks a plain-link hit (0.3) only *among non-FTS results*; a strong FTS body match (~1.0) can still rank higher. Coherent; comment added.
- **L1**: dropped the dead `entityHit` field (written, never read).

## Verification

188 tests (180 + 8 new), tsc clean. Smoke (heredoc): relations parsed, 3 relation targets stubbed, body verbatim, `recall --relation joy` returns full relations map + tags. C1 repro (leading-blank + `--mood`) now passes: remember and index agree.
