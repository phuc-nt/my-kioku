# v1.1 Phase 5 — Telegram-backup migration (tags + recursive)

**Date:** 2026-06-27 · **Commit:** b94b912 · **Branch:** feat/v1.1-emotional-relations

## What shipped

Extend `import --from-kioku-lite` to ingest the real Telegram backup: `# Kioku —` heading, per-block Python-list `tags:`, recursive folder scan.

- Extracted pure parsers to `import-kioku-lite-parser.ts` (command file was at the 200-LOC limit).
- `parseTagList`: lenient Python-list parser (`['a', "b"]` → strings).
- Import prepends a `tags::` inline line (added line; body verbatim); tags get indexed.
- Recursive scan (decision #4): one command ingests subfolders; non-diary `.md` (README) ignored.
- Hash over ORIGINAL text → idempotent even with the tags prepend.

## Key decisions / learnings

- **Real-data validation drove a data-loss fix.** First real run: `442 blocks / created=155 / bad=8`. The 8 bad blocks had **partial `event_time`** values — `"2020"`, `"2025-02"`, even `"lớp 2"` (free text). `resolveDateTime` required a full `YYYY-MM-DD` and dropped them. Fix: fall back to the `time` timestamp's date (event_time is enrichment, not a requirement). Re-run: **bad=0**, no memory silently lost. (This is exactly why migration must be tested on real data, not just fixtures.)
- **Code review (red-team, irreversible migration) verified the two hardest invariants on real append code:**
  - VERBATIM: body byte-identical, `tags::` is a prepended line. Held.
  - IDEMPOTENCY: hash over `b.text` (original), not the tags-augmented text. Re-runs create 0.
- **H1 (documented, user decision = document-not-code):** v1.0 shipped `import` WITHOUT tags. A vault already imported under v1.0 has block hashes logged → a v1.1 re-import skips them, so tags aren't backfilled. Idempotency is by-design (text-hash dedup). Fix path: import into a FRESH vault (do NOT clear the log on a populated vault — that duplicates). Documented in the manual checklist + risk table.
- **M1/M2 documented**: textual-duplicate tag-sets aren't unioned (first wins); the leading-field zone would consume a field-shaped first body line (latent, real prose data doesn't trigger).
